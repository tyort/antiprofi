# Antiprofi

## Запуск приложения через Docker

В проекте настроен запуск с использованием Docker и Docker Compose. Приложение собирается в режиме `standalone` (оптимизированная сборка Next.js), что позволяет получить минимальный размер итогового образа.

### 1. Требования
Перед началом убедитесь, что на вашем сервере или локальной машине установлены:
- [Docker](https://docs.docker.com/get-docker/)
- [Docker Compose](https://docs.docker.com/compose/install/)

### 2. Настройка окружения
Для корректной работы приложения (например, для отправки писем через форму обратной связи) требуется файл с переменными окружения.

1. Создайте файл `.env` в корне проекта (если его еще нет).
2. Убедитесь, что в нем прописаны все необходимые переменные (например, доступы к SMTP для почты).

### 3. Запуск приложения

Откройте терминал в корне проекта и выполните команду:

```bash
docker compose up -d --build
```

- Флаг `-d` (detach) запускает контейнеры в фоновом режиме, освобождая терминал.
- Флаг `--build` гарантирует, что Docker соберет свежий образ на основе последних изменений в коде.

После завершения сборки приложение будет доступно по адресу: **http://localhost:3000**

### 4. Управление контейнерами

**Просмотр логов:**
Если вам нужно посмотреть логи приложения (например, для отладки ошибок):
```bash
docker compose logs -f web
```
*(Флаг `-f` позволяет следить за логами в реальном времени. Для выхода нажмите `Ctrl+C`).*

**Остановка приложения:**
Чтобы остановить работающие контейнеры и удалить сеть:
```bash
docker compose down
```

**Перезапуск после изменения кода или `.env` файла:**
Если вы внесли изменения в исходный код или обновили переменные в `.env`, необходимо пересобрать образ:
```bash
docker compose down
docker compose up -d --build
```

### Дополнительная информация об архитектуре сборки
- **Dockerfile**: Использует многоэтапную сборку (multi-stage build) на базе `node:alpine`. Это позволяет не переносить исходный код и все `node_modules` в финальный образ, оставляя только минимально необходимые скомпилированные файлы.
- **next.config.mjs**: Включена опция `output: 'standalone'`, которая автоматически копирует только нужные файлы для запуска production-сервера, что сильно экономит место.
